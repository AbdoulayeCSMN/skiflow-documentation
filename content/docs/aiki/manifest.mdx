# Aiki.toml — Manifeste

Le fichier `Aiki.toml` est le manifeste d'un projet Harzourou. Il est créé par `aiki init` et mis à jour par `aiki add` / `aiki remove`.

## Structure complète

```toml
[package]
name        = "mon-projet"
version     = "0.1.0"
description = "Un projet Harzourou"
authors     = ["Votre Nom <vous@exemple.com>"]
license     = "MIT"
entry       = "main.harz"   # point d'entrée (défaut : main.harz)

[dependencies]
math = "1.0.0"
fmt  = "^1.0"

[dev_dependencies]
assert = "1.0.0"

[scripts]
start   = "harz main.harz"
test    = "harz test.harz"
dev     = "harz main.harz --interpret"

[build]
output = ".aiki/build"   # dossier de sortie (défaut : .aiki/build)
flags  = []              # flags supplémentaires passés à harz

[harzourou]
# compiler     = "/usr/local/bin/harz"         # chemin du compilateur
# modules_path = "/usr/local/lib/HarzourouModules"  # chemin stdlib
```

---

## `[package]`

| Champ | Requis | Description |
|-------|--------|-------------|
| `name` | ✅ | Nom du paquet (lettres minuscules, chiffres, tirets) |
| `version` | ✅ | Version SemVer (`0.1.0`) |
| `description` | — | Description courte |
| `authors` | — | Liste d'auteurs |
| `license` | — | Licence SPDX (`MIT`, `Apache-2.0`…) |
| `entry` | — | Fichier `.harz` principal (défaut : `main.harz`) |

---

## `[dependencies]` et `[dev_dependencies]`

### Forme simple (chaîne de version)

```toml
[dependencies]
math = "1.0.0"      # version exacte
fmt  = "^1.0"       # compatible 1.x
algo = "latest"     # dernière version
```

### Forme détaillée (table)

```toml
[dependencies]
math = { version = "1.0.0" }
utils = { version = "2.0.0", optional = true }
local_pkg = { version = "1.0.0", path = "../mes-utils" }
```

### Contraintes de version

| Syntaxe | Signification |
|---------|---------------|
| `"1.0.0"` | Exactement 1.0.0 |
| `"^1.0"` | Compatible 1.x.y (≥ 1.0.0, < 2.0.0) |
| `"~1.2"` | Compatible 1.2.y (≥ 1.2.0, < 1.3.0) |
| `"latest"` / `"*"` | Dernière version disponible |

## `[scripts]`

Les scripts sont des commandes shell. Le token `harz` est automatiquement remplacé par le chemin réel du compilateur.

```toml
[scripts]
start   = "harz main.harz"
test    = "harz test.harz"
bench   = "harz bench.harz --no-opt-bc --stats"
lint    = "harz main.harz --tokens"
```

Exécution :

```bash
aiki run start             # harz main.harz
aiki run start --no-opt    # harz main.harz --no-opt
aiki run test              # harz test.harz
```

<Callout type="info">
  Les flags `--interpret`, `--no-opt`, `--no-opt-bc` et `--stats` passés à `aiki run` sont injectés directement dans l'invocation de `harz`, sans modifier `Aiki.toml`.
</Callout>

## `[build]`

```toml
[build]
output = ".aiki/build"     # dossier de sortie
flags  = ["--no-opt-bc"]   # flags harz toujours actifs au build
```

---

## `[harzourou]`

Surcharge locale de la configuration du runtime :

```toml
[harzourou]
compiler     = "/home/user/harzourou/build/harz"
modules_path = "/home/user/harzourou/HarzourouModules"
```

Ces valeurs ont priorité sur `HARZOUROU_PATH` et la configuration globale `~/.aiki/config.toml`.

## Aiki.lock

Le fichier `Aiki.lock` est généré automatiquement. **Il doit être committé** dans le dépôt git pour garantir des builds reproductibles.

```json
{
  "version": 1,
  "generated_at": "2026-05-10T12:00:00Z",
  "packages": [
    {
      "name": "math",
      "version": "1.0.0",
      "source": { "type": "registry", "url": null },
      "checksum": null,
      "install_path": ".aiki/packages/math@1.0.0"
    }
  ]
}
```
