# Fonctions

Les fonctions sont déclarées avec le mot-clé `fun`. Elles supportent les paramètres typés, les valeurs par défaut, les arguments nommés et les closures.

## Syntaxe de base

```harzourou
fun nom(param1 : type, param2 : type) : type_retour {
  // corps
  return valeur
}
```

### Exemple simple

```harzourou
fun addition(a : int, b : int) : int {
  return a + b
}

flow résultat = addition(3, 5)  // 8
fprintln(résultat)
```

## Valeurs par défaut

```harzourou
fun saluer(nom : string, salut : string = "Bonjour") : string {
  return salut ~ ", " ~ nom ~ "!"
}

fprintln(saluer("Amine"))            // Bonjour, Amine!
fprintln(saluer("Sara", "Bonsoir"))  // Bonsoir, Sara!
```

## Arguments nommés

```harzourou
fun créer_point(x : float, y : float, z : float = 0.0) {
  fprintln("(" ~ str(x) ~ ", " ~ str(y) ~ ", " ~ str(z) ~ ")")
}

créer_point(x: 1.0, z: 3.0, y: 2.0)  // ordre libre → (1.0, 2.0, 3.0)
```

## Fonctions d'ordre supérieur

Harzourou supporte les fonctions comme valeurs de première classe :

```harzourou
fun appliquer(f, liste) {
  return map(liste, f)
}

flow doubles = appliquer(fun(x) { return x * 2 }, [1, 2, 3, 4])
fprintln(doubles)   // [2, 4, 6, 8]
```

## Closures

```harzourou
fun multiplicateur(n : int) {
  return fun(x : int) {
    return x * n
  }
}

flow fois3 = multiplicateur(3)
fprintln(fois3(7))   // 21
fprintln(fois3(10))  // 30
```

## `map`, `filter`, `fold`

Ces fonctions d'ordre supérieur sont natives :

```harzourou
flow nums = [1, 2, 3, 4, 5, 6]

// map — transformer chaque élément
flow carrés = map(nums, fun(x) { return x * x })
fprintln(carrés)   // [1, 4, 9, 16, 25, 36]

// filter — garder selon condition
flow pairs = filter(nums, fun(x) { return x % 2 == 0 })
fprintln(pairs)    // [2, 4, 6]

// fold — réduire à une valeur
flow somme = fold(nums, 0, fun(acc, x) { return acc + x })
fprintln(somme)    // 21
```

## Opérateur pipe `|>`

`x |> f(args)` insère `x` comme premier argument de `f` :

```harzourou
flow résultat = [1, 2, 3, 4, 5, 6]
    |> filter(fun(x){ return x % 2 == 0 })   // [2, 4, 6]
    |> map(fun(x){ return x * x })            // [4, 16, 36]
    |> fold(0, fun(acc, x){ return acc + x }) // 56

fprintln(résultat)  // 56
```

## Composition `>>`

`f >> g` crée une closure `fun(x){ return g(f(x)) }` :

```harzourou
fun increment(x){ return x + 1 }
fun square(x)   { return x * x }
fun double(x)   { return x * 2 }

flow pipeline = increment >> square >> double
// pipeline(4) = double(square(5)) = double(25) = 50

fprintln(pipeline(4))   // 50
```

## Unités (`unit`)

Les `unit` sont des blocs fonctionnels avec entrées/sorties explicites, typiques des architectures de flux de données :

```harzourou
import list

unit normaliseur(in: données, out: résultat) {
  flow min_val = list.minimum(données)
  flow max_val = list.maximum(données)
  flow range   = max_val - min_val
  résultat = map(données, fun(x) { return (x - min_val) / range })
}
```

<Callout type="info">
  Les `unit` permettent de composer des pipelines de traitement de données de façon déclarative.
</Callout>

## Gestion des erreurs

```harzourou
fun diviser(a : float, b : float) : float {
  if b == 0.0 {
    throw "Division par zéro"
  }
  return a / b
}

try {
  flow r = diviser(10.0, 0.0)
  fprintln(r)
} catch e {
  fprintln("Erreur : " ~ e)
} finally {
  fprintln("Opération terminée")
}
```

## Application partielle

```harzourou
fun multiply(a, b){ return a * b }

flow times3 = partial(multiply, 3)

fprintln(times3(7))   // 21
fprintln(times3(10))  // 30
```

## `flip` — inversion d'arguments

```harzourou
fun subtract(a, b){ return a - b }
flow flipped = flip(subtract)

fprintln(flipped(3, 10))   // 10 - 3 = 7
```
