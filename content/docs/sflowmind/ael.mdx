# Module `ael` — Adaptive Execution Lattice

L'**AEL** est un treillis de décision adaptatif combinant :
- **MCTS / UCT** — sélection par Upper Confidence Bound,
- **ε-greedy** et **Softmax** (Boltzmann) comme stratégies alternatives,
- **Q-learning** tabulaire par nœud,
- **rollouts** Monte Carlo,
- apprentissage **fédéré multi-agents** avec consensus,
- élagage (pruning) des nœuds peu visités.

## Import

```skiflow
import ael
```

---

## Types de nœuds (`op_type`)

| Valeur | Nom | Rôle |
|--------|-----|------|
| 0 | `AEL_OP_START` | Point d'entrée unique du treillis |
| 1 | `AEL_OP_COMPUTE` | Transformation / calcul générique |
| 2 | `AEL_OP_BRANCH` | Branchement conditionnel (fan-out) |
| 3 | `AEL_OP_CALL` | Invocation récursive de sous-treillis |
| 4 | `AEL_OP_END` | État terminal |

---

## Politiques de sélection (`policy_mode`)

| Valeur | Mode | Description |
|--------|------|-------------|
| 0 | UCT | Upper Confidence Bound (par défaut, comportement v1) |
| 1 | ε-greedy | Exploitation + exploration uniforme avec probabilité ε |
| 2 | Softmax | Distribution Boltzmann sur les valeurs Q (température τ) |

---

## Cycle de vie

### `ael.new(policy_mode, rate)` -> lattice

Crée un treillis avec la politique choisie. `rate` = ε (ε-greedy) ou τ (softmax).

```skiflow
flow L = ael.new(0, 0.1)   // UCT, rate ignoré
flow L = ael.new(1, 0.15)  // ε-greedy, ε = 0.15
flow L = ael.new(2, 1.0)   // Softmax, τ = 1.0
```

### `ael.add(L, op_type)` -> node_id

Ajoute un nœud et retourne son identifiant stable.

```skiflow
flow start   = ael.add(L, 0)   // START
flow compute = ael.add(L, 1)   // COMPUTE
flow branch  = ael.add(L, 2)   // BRANCH
flow end_a   = ael.add(L, 4)   // END
flow end_b   = ael.add(L, 4)
```

### `ael.root(L, node_id)` -> null

Désigne le nœud racine du treillis.

```skiflow
ael.root(L, start)
```

---

## Construction du treillis

### `ael.connect(L, from, to, prior, cost)` -> null

Ajoute un arc probabiliste. `prior` ∈ \[0,1\] = probabilité a priori, `cost` ≥ 0.

```skiflow
ael.connect(L, start,   compute, 1.0, 0.0)
ael.connect(L, compute, branch,  0.9, 0.5)
ael.connect(L, branch,  end_a,   0.6, 1.0)
ael.connect(L, branch,  end_b,   0.4, 1.5)
```

---

## Sélection & exploration

### `ael.select(L, node_id, rng)` -> next_id

Sélectionne le prochain nœud depuis `node_id` selon la politique active.
`rng` ∈ \[0,1\] = valeur aléatoire (fournie par l'appelant).

```skiflow
flow next = ael.select(L, start, rand())
```

### `ael.rollout(L, from, to, discount, rng)` -> null

Exécute un rollout Monte Carlo de `from` à `to`, actualisant les valeurs Q
avec le facteur d'actualisation `discount` ∈ (0, 1\].

```skiflow
ael.rollout(L, start, end_a, 0.95, rand())
```

---

## Retour d'expérience

### `ael.reward(L, node_id, r)` -> null

Met à jour la récompense accumulée du nœud `node_id` avec la valeur `r`.

```skiflow
ael.reward(L, end_a, 1.0)
ael.reward(L, end_b, 0.3)
```

### `ael.update(L, discount)` -> null

Propage les mises à jour de politique globale (actualise la baseline EWMA).

```skiflow
ael.update(L, 0.95)
```

---

## Statistiques par nœud

```skiflow
ael.node_count(L)             // nombre de nœuds
ael.node_value(L, node_id)    // Q-value estimée
ael.node_visits(L, node_id)   // nombre de visites
ael.node_uncertainty(L, node_id)  // 1/√(visits)
```

---

## Maintenance

### `ael.prune(L, min_visits)` -> null

Marque comme "pruné" les nœuds avec `visit_count < min_visits`.

```skiflow
ael.prune(L, 5)   // élague les nœuds visités moins de 5 fois
```

---

## Q-learning tabulaire

### `ael.rl_init(L, lr, gamma)` -> null

Initialise le moteur Q-learning (taux d'apprentissage `lr`, facteur d'actualisation `gamma`).

```skiflow
ael.rl_init(L, 0.1, 0.95)
```

### `ael.q_update(L, node_id, reward, next_id)` -> null

Mise à jour Q(s,a) ← Q(s,a) + lr × (r + γ × max_a' Q(s',a') − Q(s,a)).

```skiflow
ael.q_update(L, compute, 0.8, branch)
```

### `ael.q_greedy(L, node_id)` -> next_id

Sélectionne le successeur avec la Q-value maximale (pure exploitation).

```skiflow
flow best = ael.q_greedy(L, start)
```

### `ael.q_value(L, node_id)` -> float

Lit la Q-value courante d'un nœud.

### `ael.record_step(L, from, to, reward)` -> null

Enregistre une transition (from, to, reward) dans l'historique du replay.

### `ael.reinforce(L, node_id, reward)` -> null

Applique une mise à jour de type REINFORCE (policy gradient simple).

---

## Apprentissage fédéré multi-agents

<Callout type="info">
  Le mode multi-agents permet à plusieurs treillis indépendants de partager leurs connaissances via consensus.
</Callout>

### `ael.multi_init(L, n_agents)` -> null

Initialise les structures multi-agents pour `n_agents` agents.

### `ael.multi_update(L, agent_id, node_id, reward)` -> null

Met à jour localement la Q-value de l'agent `agent_id`.

### `ael.multi_select(L, agent_id, node_id, rng)` -> next_id

Sélection avec politique locale à l'agent.

### `ael.multi_consensus(L, weights)` -> null

Agrège les politiques de tous les agents par moyenne pondérée.
`weights` = liste de floats (un poids par agent).

```skiflow
flow poids = [0.4, 0.3, 0.3]
ael.multi_consensus(L, poids)
```

### `ael.multi_record(L, agent_id, from, to, reward)` -> null

Enregistre une transition pour un agent.

### `ael.multi_reinforce(L, agent_id, node_id, reward)` -> null

REINFORCE pour un agent spécifique.

### `ael.multi_joint(L)` -> null

Calcule la politique jointe entre tous les agents.

### `ael.multi_best(L, node_id)` -> next_id

Retourne le meilleur successeur selon la politique jointe.

ael.save(L, "lattice.aidb")

## Exemple complet — navigation adaptative

```skiflow
import ael

// Treillis de décision : planification de chemin
flow L = ael.new(0, 0.1)   // UCT

flow s0 = ael.add(L, 0)   // START
flow s1 = ael.add(L, 1)   // COMPUTE : évaluation état
flow s2 = ael.add(L, 2)   // BRANCH  : choix de direction
flow s3 = ael.add(L, 1)   // COMPUTE : exécution action A
flow s4 = ael.add(L, 1)   // COMPUTE : exécution action B
flow s5 = ael.add(L, 4)   // END : succès
flow s6 = ael.add(L, 4)   // END : échec

ael.root(L, s0)
ael.connect(L, s0, s1, 1.0, 0.0)
ael.connect(L, s1, s2, 1.0, 0.2)
ael.connect(L, s2, s3, 0.6, 0.5)
ael.connect(L, s2, s4, 0.4, 0.5)
ael.connect(L, s3, s5, 0.8, 1.0)
ael.connect(L, s3, s6, 0.2, 1.0)
ael.connect(L, s4, s5, 0.4, 1.5)
ael.connect(L, s4, s6, 0.6, 1.5)

// Q-learning
ael.rl_init(L, 0.1, 0.95)

// 100 épisodes d'apprentissage
fun episode(k) {
    ael.rollout(L, s0, s5, 0.95, rand())
    flow result = ael.q_greedy(L, s2)
    flow r = result == s3 ? 1.0 : 0.3
    ael.q_update(L, s2, r, result)
}

flow episodes = func.range(0, 99)
each(episodes, fun(k){ episode(k) })

fprintln "Q(s2->s3) = " ~ to_string(ael.q_value(L, s3))
fprintln "Meilleur choix : " ~ to_string(ael.q_greedy(L, s2))

ael.save(L, "planner.aidb")
```
