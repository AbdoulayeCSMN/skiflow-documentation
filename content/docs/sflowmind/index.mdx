# SFlowMind

**SFlowMind** est la bibliothèque d'IA embarquée de SkiFlow — un ensemble de modules C++ purs (zéro dépendance externe) accessibles directement depuis le langage SkiFlow via l'instruction `import`.
Le code source de cette bibliothèque est disponible sur (github)[https://github.com/AbdoulayeCSMN/SkiFlowStructurePieces]
> Développé par C. Abdoulaye — ENSAM Meknès, filière IATD-SI.

## Paradigme commun

Tous les modules SFlowMind suivent le même patron :

```skiflow
// 1. Créer une instance
flow g = nsmg.new(128)

// 2. Construire la structure
flow noeud_a = nsmg.add(g, "concept_A")
flow noeud_b = nsmg.add(g, "concept_B")
nsmg.link(g, noeud_a, noeud_b, 0.8, 2)

// 3. Calculer
nsmg.activate(g, noeud_a, 1.0)
nsmg.spread(g, noeud_a)

// 4. Lire les résultats
fprintln nsmg.activation(g, noeud_b)

// 5. Persister
nsmg.save(g, "graph.aidb")
```

## Intégration fonctionnelle

SFlowMind s'intègre naturellement avec le cœur fonctionnel de SkiFlow :

```skiflow
import nsmg
import func

flow g = nsmg.new(64)
flow concepts = ["chien", "chat", "oiseau", "poisson", "reptile"]

// Ajouter tous les nœuds avec map
flow ids = map(concepts, fun(label){ return nsmg.add(g, label) })

// Pipeline d'analyse
flow n_nodes = nsmg.node_count(g)
flow activations = func.range(0, n_nodes - 1)
    |> map(fun(i){ return nsmg.activation(g, i) })
    |> filter(fun(a){ return a > 0.1 })

fprintln "Nœuds actifs : " ~ to_string(activations)
```

## Format de persistance `.aidb`

Tous les modules supportent `save` / `load` vers un fichier binaire `.aidb` (AI DataBase) :

```skiflow
nsmg.save(g,   "knowledge.aidb")
flow g2 = nsmg.load("knowledge.aidb")
```

<Callout type="warning">
  Le format `.aidb` est spécifique à SFlowMind v2. Les fichiers produits par une version antérieure ne sont pas garantis compatibles.
</Callout>
